### [Op] 2025/4/25 16:51 ~ 17:05 攻擊事件

**時間:**

*   事件約於 **2025-04-25 16:51 (UTC+8)** 開始，系統監控 (`Cofacts monitor 🚨`) 發出 `cofacts.tw` 因 HTTP timeout 而 Unhealthy 的警報。
*   同時，`mrorz` 觀察到伺服器負載 (load) 出現異常。

**狀況:**

*   `cofacts.tw` 服務因 HTTP timeout 而無法正常回應。 ![](https://g0v.hackmd.io/_uploads/Bk2noChyle.png)
*   伺服器負載過高。 ![](https://g0v.hackmd.io/_uploads/SJ76sRnJel.png)


**來源分析 (根據討論):**

*   **主要流量高峰:** 來自**新加坡** IP 的流量出現顯著 spike，時間點與服務中斷吻合。![](https://g0v.hackmd.io/_uploads/BkxbJhR3Jel.png)
*   **可疑 ASN:** `nonumpa` 觀察到 ASN 在短時間內使用多個不同 IP 連接網站。 ![](https://g0v.hackmd.io/_uploads/HywAjR2Jex.png)
*   **台灣 IP:** `nonumpa` 發現有 5 個來自**台灣 (中華電信 ASN)** 的 IP 在稍早時間 (具體時間點未明) 有異常行為，以固定 2-3 秒的頻率發送請求。
* ![](https://g0v.hackmd.io/_uploads/HJu5A1p1xg.png)
![](https://g0v.hackmd.io/_uploads/BygXPAyT1lg.png)
*   **WAF Skip 流量:** `mrorz` 初始觀察到大量被 WAF 規則 `skip` (跳過檢查) 的請求，流量來源似乎來自 Google 的 ASN，造成一些困惑。

**處理與緩解:**

*   `mrorz` 於 **2025-04-25 17:05 (UTC+8)** 左右，關閉了 WAF (Web Application Firewall) 中兩條設定為 `skip` 的規則，停止豁免這些請求。

**結果:**

*   在調整 WAF 規則後，系統隨即恢復正常運作。
