### 重點觀察與建議

1. **6/29 CPU 異常偏高**：Elasticsearch 和 API server 在 6/29 皆有明顯 CPU 峰值，之後恢復正常。可能是週日流量較高或 Cloudflare cache miss 觸發大量 origin 請求。

2. **Swap 緩慢上升（值得關注）**：Swap avg 從 6/30 的 42.1% 爬升至 7/06 的 43.8%，一週漲約 1.7%。雖仍在安全範圍，但若持續爬升需注意（主因仍為 Elasticsearch 極大記憶體佔用）。

3. **磁碟使用率 72%**：目前 49G/67G。不緊急，但建議持續監控，若資料量持續增長需規劃擴容。

4. **site-tw Memory 接近 1 GiB**：Cloud Run p99 記憶體達 1.01 GiB，建議確認容器 memory limit 設定，避免 OOM。

5. **WAF 效果觀察**（egress 降費任務）：site-tw 本週每日流出 4–7 GB，6/29 有 8.2 GB 突波。7 月帳單驗收時可對比 WAF 上線前後的 egress 費用，觀察 WAF 封鎖 SEO 爬蟲的實際效果。

6. **所有 Cloud Run 服務正常運作**，無異常停機或部署失敗。`cofacts-ai` 於 7/04 有一次重新部署。
