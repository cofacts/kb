## Setup Dev environments in GGM & Lucien's laptop

安裝時遇到的問題：

* 目前 `rumors-api` 設計為只要有 `docker-compose` 就能用
* 但由於 `yarn install` 與 server 第一次一定會失敗（docker-compose.yml 的 [`dependes_on` 不會等待 ready](https://docs.docker.com/compose/compose-file/#dependson)），其實對開發者並不友善
* `USER_ID=$UID` 令開發者追問這是在幹嘛的，但其實只是為了讓 yarn install 可以用自己而非 root 的 UID 來安裝。或許直接讓開發者在 docker 之外自己 yarn install 更直覺。
* 原本為了讓沒裝 node 的人也能開發，但看起來造成的困擾似乎比安裝 node + yarn 本身還麻煩。

另外花了一些時間在講解 GraphQL。或許需要早日完成 [GraphQL 文件](https://hackmd.io/s/B1sa6Luj)。


結論：
1. 改寫 rumors-api 的 `README.md`，引導開發者先安裝 node 再來開發。
// Lucien 覺得把 yarn 移到 docker 之外就好，讓 developer `yarn` 完再 `docker up`
2. docker-compose 只用來同時開啟 db、kibana 與 elasticsearch。
