### Line bot: Flex message update

Discussion in PR: https://github.com/cofacts/rumors-line-bot/pull/93

Hazel:
會按「重新填寫理由」是什麼狀況？

大部分應該是使用者傳了新的進來

但他們這個時候應該都是想查新的。

Orz:
Hmm 但這要用 LIFF 解。
那這個部分就先照現況好了


:::success
### 結論

Flex message is a 簡便解法, let's keep it simple. No changes needed.

#### Reason

此步驟會有問題，大部分是使用者「錯傳新訊息進來」。但若要根除此問題，就應該用更複雜的 LIFF。

現階段應該只是先試試看把送出訊息流程用 flex message 呈現之後，是否可以降低誤傳到可以接受的程度，因此不應該把流程搞得太複雜，故保留「好的，請重新填寫理由」為佳。
:::
