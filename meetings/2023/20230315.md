> https://lookerstudio.google.com/reporting/18J8jZYumsoaCPBk9bdRd97GKvi_W5v-r/page/NrUQ
> LINE BOT的熱門詢問很多是以前有人回過的，但佐證可能失連或過時
> 要不要每週開會時檢查前七天熱門中有回應的查核，連結有失連否？
> 或是小聚時每人分一則檢查連結有無失連或查核答案有無過時（日期範圍可能要討論）
> [name=cai]

- 現在有 crawler 但成功率很低 [name=orz]
- 但 crawler 就是為了保存這些連結，這樣做應該就可以了 [name=bil]
- 有些連結失聯就失聯了，不一定有可用的 alternative [name=bil]
- 事後檢查要消耗比較多人力 [name=bil]
- 只挑熱門的回應來檢查 [name=bil]
- 每週可以檢查前五，但有點難解決失聯問題 [name=bil]
  - 小聚比較忙而且兩個月一次
- 理想狀態是 line使用者有人發現錯誤然後自己來寫新的查核，但實際上並不會😅 [name=cai]
  - 會按 downvote 啦 XD

### [Infra] Linode to Cloud run

RAM / CPU usage on production machine

- Elasticsearch: 9GB RAM, 41% CPU (out of 6 cores)
![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_b231064ee0ac156a2bbd2084789cf797.png)
- Website: 2GB RAM, 42% CPU (out of 6 cores) ![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_f0db6d53db70b0d04b24e533ccc3bfbe.png)
- API & LINE bot (both src/www/build/index, cannt differentiate): 1GB RAM, 18% CPU (out of 6 cores) ![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_c66ab7924448f4dc1bea403401b0e131.png)
